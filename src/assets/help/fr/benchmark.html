Comparer

Cette commande permet de comparer les performances de compression et décompression RAR sur différentes plates-formes matérielles. Les versions identiques de WinRAR et RAR pour Android partagent essentiellement le même code de base de référence. Seule une partie de l'interface de comparaison sous Android utilise Java. Les opérations intensives sur le processeur sont effectuées en code natif, ainsi la performance de la machine virtuelle Java n'est pas mesurée.

La première étape de l'algorithme génère un flux de nombres aléatoires avec une relative faible redondance, proche des conditions permettant les moins bonnes performances de RAR. Ce flux est passé à travers les algorithmes de compression et décompression de RAR. La décompression obtenue est comparée avec les données d'origine. En cas de différence trouvée, "Erreurs trouvées: Oui" s'affiche. Ces erreurs peuvent venir de problèmes matériels comme la fiabilité de la mémoire. Cependant, n'attendez pas de cette commande qu'elle remplace des logiciels de diagnostic spécialisés. Son but est la comparaison, pas le test du matériel.

La fenêtre de commande contient également la taille des données traitées, le temps écoulé et la vitesse de compression, en cours et une fois terminée, en kilo-octets par seconde. La vitesse résultante est calculée en tant que vitesse maximale pour la période de surveillance définie et n'est pas mise à jour une fois fixée. Le temps pour déterminer la vitesse résultante peut s'avérer plus long pour un matériel plus lent.

La commande Comparer utilise la version multi-tâches de compression de RAR et les implémentations mono tâche des autres algorithmes, comme pour la décompression et la génération de nombres aléatoires. La compression étant source de plus de recours au processeur, la performance obtenue dépend sensiblement du nombre de cœurs de processeur disponibles. Par défaut, tous les cœurs disponibles sont utilisés, mais si vous souhaitez voir les résultats pour un seul cœur ou un nombre limité de cœurs de processeurs, vous pouvez appuyer sur le champ "nombre de tâches" pour redémarrer les mesures avec un autre nombres de tâches.

Il n'est pas garanti que différentes versions de RAR obtiennent les mêmes valeurs de vitesse de compression. Si vous utilisez cette commande pour comparer la performance de différents périphériques, assurez-vous que vous exécutez la même version de RAR dans tous les tests.

La commande Comparer ne se termine jamais automatiquement, vous devez y mettre fin manuellement. Elle n'utilise pas d'enregistrement permanent, toutes les opérations sont effectuées dans la mémoire.
